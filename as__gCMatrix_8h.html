Create a *gCMatrix from a triplet-formatted store of non-zero entries. Best used when the number of non-zero entries is not known in advance.

Template Parameters
VAn Rcpp::Vector class, to be used as the x slot in the output *gCMatrix. Only Rcpp::NumericVector and Rcpp::LogicalVector are supported.
TType of data in the triplet store. It should be possible to cast this to the storage type of V.
Parameters
nrNumber of rows.
ncNumber of columns.
holderTriplet-formatted information for all non-zero entries. The key should contain the zero-based column index (first) and the zero-based _row__index (second). The value should contain the non-zero value itself.
Returns
A dgCMatrix or lgCMatrix instance (depending on V) containing all entries in holder.

Create a *gCMatrix by modifying the non-zero values (but not their order or position) in an existing *gCMatrix.

Template Parameters
VAn Rcpp::Vector class, to be used as the x slot in the output *gCMatrix. Only Rcpp::NumericVector and Rcpp::LogicalVector are supported.
TType of data in the triplet store. It should be possible to cast this to the storage type of V.
Parameters
oldA *gCMatrix object. This does not need to be of a type corresponding to V.
xA vector of non-zero values to use to replace the existing x slot in old. The row index and column identity oeach value is assumed to be the same as the non-zero values already in old.
Returns
A dgCMatrix or lgCMatrix instance (depending on V) containing indexing information from old but updated with the non-zero values in x.

template<class V >
Rcpp::S4 beachmat::generate_gCMatrix ( )

Generate a *gCMatrix of the appropriate class for the specified Rcpp::Vector class in V.

Note
This is an internal function and should not be called directly by beachmat users.
Template Parameters
VAn Rcpp::Vector class. Currently only Rcpp::NumericVector and Rcpp::LogicalVector are supported.
Returns
An Rcpp::S4 object of the appropriate *gCMatrix class for V, either dgCMatrix or lgCMatrix for double-precision and logical data, respectively. All other choices of V will trigger a compile-time error.
